Comprensión del Algoritmo

El algoritmo procesa los vértices uno a uno en orden v₁, v₂, …, vₙ. Para cada vértice vᵢ:

  • ①Reinicia disponibilidad: marca los colores 1…colores-usados como disponibles.
  • ②Revisa vecinos anteriores: para todo vⱼ ya coloreado (j < i) adyacente a vᵢ, marca color[vⱼ] como no disponible.
  • ③Asigna el mínimo color disponible: toma el color c más pequeño que no esté bloqueado.
  • ④Nuevo color si necesario: si ningún color existente está libre, crea uno nuevo incrementando colores-usados.
💡Clave greedy: la elección es localmente óptima (el color más pequeño posible para vᵢ) pero no garantiza el óptimo global (χ(G) mínimo). Sin embargo, sí garantiza usar a lo sumo Δ(G)+1 colores.

Cuando el algoritmo procesa vᵢ, solo los vecinos ya coloreados (con j < i) pueden bloquear colores. El número de vecinos ya coloreados de vᵢ es como máximo deg(vᵢ) ≤ Δ(G). Por tanto:

Si deg(vᵢ) ≤ Δ(G) vecinos bloquean colores,
entonces a lo sumo Δ(G) colores están bloqueados.

{1, 2, …, Δ(G)+1} tiene Δ(G)+1 colores
⟹ siempre queda al menos 1 color disponible en ese rango
⟹ nunca se necesita el color Δ(G)+2
⚠️Importante: el algoritmo mira solo vecinos anteriores (j < i), no todos. Pero como el grado total de vᵢ es ≤ Δ(G), sus vecinos anteriores también son ≤ Δ(G).

El mecanismo es el arreglo disponible[]. El proceso funciona así:

Para cada vecino vⱼ (j < i) de vᵢ:
  if color[vⱼ] ≠ 0:
    disponible[color[vⱼ]] ← false ← bloquea ese color

Resultado: disponible[c] = true ⟺ ningún vecino anterior usa el color c

Al seleccionar el menor c con disponible[c] = true, se garantiza que ningún vecino ya procesado tiene ese color. Los vecinos futuros (j > i) aún no tienen color, así que no hay conflicto posible con ellos en este momento.

🔑La invariante I3 captura exactamente esto: tras cada iteración, el coloreo parcial mantiene que vértices adyacentes ya coloreados tienen colores distintos.

Invariantes de Bucle

I0 — Rango contiguo
Los colores usados forman el rango {1, 2, …, colores-usados} sin huecos.
colores-usados = |{c : ∃v, color[v]=c}|
I1 — Coloreo válido parcial
Para todo j < i, color[vⱼ] está definido y no conflictivo con sus vecinos anteriores.
∀j < i: color[vⱼ] ∈ {1,…,colores-usados}
I2 — Cota Δ+1
Los colores usados nunca superan el grado máximo del grafo más uno.
colores-usados ≤ Δ(G) + 1
I3 — Propiedad de coloreo
El coloreo parcial es válido: adyacentes coloreados tienen colores distintos.
∀(vⱼ,vₖ)∈E, j,k < i: color[vⱼ] ≠ color[vₖ]

Estado inicial: color[·]=0, colores-usados=0, disponible[·]=true.

  • I0El rango es {1,…,0} = ∅. Vacío trivialmente contiguo. ✓
  • I1No existe j < 1 = i inicial, por lo que la condición universal ∀j<i es verdadera por vacuidad. ✓
  • I2colores-usados = 0 ≤ Δ(G)+1 ya que Δ(G) ≥ 0. ✓
  • I3No hay vértices coloreados, así que no hay par (vⱼ,vₖ) que viole la condición. Vacuamente verdadero. ✓

Hipótesis inductiva: Al inicio de i=k, los invariantes I0–I3 se cumplen para el coloreo de {v₁,…,v_{k-1}}.

Al ejecutar la iteración para vₖ:

  • I1Lineas 5-7 (reinicio): disponible[c] vuelve a true para c=1..colores-usados. Por I0, esto cubre exactamente todos los colores existentes.
    Lineas 8-12 (bloqueo): Para cada vecino vⱼ (j<k) coloreado, disponible[color[vⱼ]] ← false. Esto no cambia color[], solo disponible[]. I1 se preserva.
  • I3El color c elegido para vₖ satisface disponible[c]=true, lo que significa que ningún vecino anterior tiene ese color. Por I3 (hip. ind.) el coloreo de {v₁,…,v_{k-1}} ya es válido. Agregar vₖ con un color distinto al de todos sus vecinos anteriores extiende I3 a {v₁,…,vₖ}. ✓
  • I2Caso A (color existente): colores-usados no cambia, I2 trivialmente preservado.
    Caso B (nuevo color): Se incrementa colores-usados en 1. ¿Por qué c > colores-usados implica que necesitábamos el nuevo color? Porque todos los colores 1..colores-usados estaban bloqueados, es decir, hay al menos colores-usados vecinos distintos de vₖ ya coloreados. Pero el número de vecinos anteriores de vₖ es ≤ deg(vₖ) ≤ Δ(G). Entonces: colores-usados_nuevo = colores-usados_viejo + 1 ≤ Δ(G) + 1 ✓
  • I0El nuevo color asignado es exactamente colores-usados (el nuevo valor). El rango {1,…,colores-usados_nuevo} sigue siendo contiguo porque solo se agrega el número colores-usados_nuevo = colores-usados_viejo + 1. No hay huecos. ✓

Estructura de la demostración:

  • ①P implica precondición del bucle: La inicialización (color[·]=0, colores-usados=0) bajo P satisface I0–I3 (demostrado en 2a).
  • ②Preservación: El paso inductivo (2b) muestra que si I0–I3 se cumplen al inicio de cualquier iteración i, se preservan al final.
  • ③Al terminar el for: i=n+1, es decir, todos los vértices han sido procesados. Por I1, todos tienen color válido. Por I3, no hay conflictos.
  • ④Q1 — Sin conflictos: I3 con i=n+1 dice que ∀(vⱼ,vₖ)∈E: color[vⱼ]≠color[vₖ]. Esto es exactamente la primera parte de Q. ✓
  • ⑤Q2 — Cota Δ+1: I2 con i=n+1 dice colores-usados ≤ Δ(G)+1. Esto es la segunda parte de Q. ✓
🎯La demostración completa de la terna de Hoare se reduce a: (a) inicialización correcta, (b) preservación de invariantes, (c) invariantes implican postcondición al terminar.

💡Para el bucle while: la cota es colores-usados − c + 1. El while termina cuando c > colores-usados (sale sin encontrar color) o disponible[c]=true (encontró color). En ambos casos la cota llega a 0.

La terminación se demuestra mostrando que cada bucle tiene una función de terminación estrictamente decreciente y acotada inferiormente:

  • ①For externo: i aumenta de 1 a n. Ejecuta exactamente n iteraciones. Termina siempre que |V|=n es finito.
  • ②For de reinicio: c aumenta de 1 a colores-usados. Como colores-usados ≤ n (finito), termina.
  • ③For de vecinos: recorre los vecinos con j < i, que son a lo sumo deg(vᵢ) ≤ |V|−1 finitos.
  • ④While del mínimo: c aumenta en 1 en cada iteración. El while tiene guarda c ≤ colores-usados, así que ejecuta a lo sumo colores-usados iteraciones ≤ Δ(G)+1 ≤ n.

Al ser G finito (|V|=n, |E|=m), todos los bucles están acotados. El algoritmo termina en tiempo finito. ∎

For externo: n iteraciones
For reinicio: O(Δ) por iteración → total O(n·Δ)
For vecinos: suma de deg(vᵢ) para i=1..n = 2|E| = 2m
While mínimo: O(Δ) en peor caso por iteración → total O(n·Δ)

⟹ T(n,m) = O(n·Δ + m)

Dado que Δ ≤ n−1, en el peor caso (grafo denso): O(n² + m). Para grafos dispersos donde m ≈ n: O(n·Δ + n).

📊Con listas de adyacencia y usando arreglos para disponible[], la complejidad es O(n + m) si se reinicia disponible[] solo para los colores usados (que son ≤ Δ+1 y ≤ n).

Límite Superior del Número Cromático

Demostración por inducción en n = |V|:

  • Basen=1: un solo vértice, χ(G)=1 ≤ Δ(G)+1 = 0+1 = 1. ✓
  • H.I.Todo grafo con n−1 vértices satisface χ ≤ Δ+1.
  • P.I.Sea G con n vértices. Tome vₙ y sea G' = G − {vₙ}. Por H.I., G' tiene un coloreo válido con ≤ Δ(G')+1 ≤ Δ(G)+1 colores. El vértice vₙ tiene deg(vₙ) ≤ Δ(G) vecinos, que usan a lo sumo Δ(G) colores distintos. Como el total de colores disponibles es Δ(G)+1 > Δ(G), queda al menos 1 color libre para vₙ. ✓

Este es exactamente el mismo argumento que usa el algoritmo greedy en cada iteración. La demostración constructiva da el algoritmo; la demostración del algoritmo da el teorema.

🔗Conexión clave: I2 (colores-usados ≤ Δ+1) es exactamente el lema central de la demostración clásica, traducido al lenguaje de los invariantes.

Teorema de Brooks (1941): Para todo grafo conexo G que no sea un grafo completo Kₙ ni un ciclo impar C_{2k+1}:

χ(G) ≤ Δ(G)

La invariante I2 dice colores-usados ≤ Δ(G)+1. Esto es una cota ligeramente más débil que el Teorema de Brooks, porque el algoritmo greedy no siempre alcanza el óptimo:

  • ●I2 garantiza que el greedy usa a lo sumo Δ+1 colores (la cota de Brooks + 1).
  • ●El Teorema de Brooks dice que χ(G) ≤ Δ(G) para grafos que no son completos ni ciclos impares.
  • ●El greedy puede usar exactamente Δ+1 colores incluso cuando el óptimo sería Δ, dependiendo del orden de los vértices.
⚠️I2 captura la cota del greedy, no el Teorema de Brooks en su versión exacta. Para alcanzar Brooks se necesitarían algoritmos más sofisticados (o una reordenación óptima de vértices).
  • 1Dependencia del orden: El número de colores usado depende del orden en que se procesan los vértices. Con distinto orden, el mismo greedy puede usar diferente número de colores.
  • 2No alcanza χ(G) en general: El coloreo mínimo (número cromático χ(G)) es NP-difícil de calcular. El greedy garantiza ≤ Δ+1, pero puede ser mayor que χ(G).
  • 3No siempre alcanza Brooks: Para grafos que no son completos ni ciclos impares, el greedy puede usar Δ+1 colores cuando bastarían Δ.
  • 4Sin backtracking: El greedy no revisa decisiones pasadas. Una mala elección temprana puede bloquear colores para vértices posteriores.
💡Dato experto: El problema de coloreo óptimo es NP-completo (demostrado por Karp, 1972). El greedy con Δ+1 colores es la mejor garantía eficiente en tiempo polinomial para grafos generales.

Preguntas Extra — Definiciones Clave

χ(G) (número cromático) = mínimo número de colores necesarios para colorear G de forma propia (vértices adyacentes con colores distintos).

Definición formal:
χ(G) = min{ k : ∃ coloreo propio de G con k colores }

Cotas importantes:

ω(G) ≤ χ(G) ≤ Δ(G)+1

donde ω(G) es el tamaño de la clique máxima (cota inferior) y Δ(G)+1 es la cota superior del greedy.

Usar disponible[] permite encontrar el mínimo color en tiempo O(Δ) en lugar de O(Δ²). Sin el arreglo auxiliar, para cada candidato de color c habría que verificar si algún vecino tiene ese color, lo que requeriría otro bucle.

Sin disponible[]: O(Δ) vecinos × O(Δ) búsqueda = O(Δ²) por vértice
Con disponible[]: O(Δ) para marcar + O(Δ) para buscar mínimo = O(Δ) por vértice

El arreglo disponible[] es una técnica de hashing implícito: usa el color como índice directo.

Preguntas Extra — Casos Extremos

Grafo completo Kₙ: Cada vértice es adyacente a todos los demás. Δ(Kₙ) = n−1. El algoritmo greedy asigna un color diferente a cada vértice (ya que al procesar vᵢ, los i−1 vecinos anteriores ya usan i−1 colores distintos). Resultado: n colores = χ(Kₙ) = Δ+1. El greedy es óptimo aquí.

Árbol (sin ciclos): Todo árbol es bipartito, luego χ(árbol) = 2 (si tiene ≥ 1 arista). El greedy asigna colores 1 y 2 alternando. Δ puede ser grande (hasta n−1 en una estrella), pero el greedy usa solo 2 colores. Aquí χ(G) ≪ Δ+1.

No, el greedy nunca usa más de Δ(G)+1 colores. La demostración directa es:

  • ①Un nuevo color solo se asigna cuando todos los colores 1..colores-usados están bloqueados.
  • ②Un color c está bloqueado ⟺ algún vecino anterior de vᵢ tiene ese color.
  • ③El número de vecinos anteriores de vᵢ es ≤ deg(vᵢ) ≤ Δ(G).
  • ④Por tanto, a lo sumo Δ(G) colores pueden estar bloqueados simultáneamente.
  • ⑤Si se necesita un nuevo color, colores-usados se incrementa en 1. Esto solo ocurre cuando colores-usados ≤ Δ(G) (pues Δ(G) vecinos bloquearon exactamente colores-usados colores, con colores-usados ≤ Δ(G)). El nuevo valor es colores-usados+1 ≤ Δ(G)+1.
⟹ colores-usados ≤ Δ(G)+1 siempre

Preguntas Extra — Comparación y Límites

La Lógica de Hoare y la inducción matemática son isomorfas para la verificación de bucles:

Inducción MatemáticaInvariante de Bucle (Hoare)
Base: P(0) verdaderoInicialización: I se cumple antes del bucle
Paso: P(k) ⟹ P(k+1)Preservación: I ∧ guarda ⟹ I tras una iteración
Conclusión: P(n) verdaderoPostcondición: I ∧ ¬guarda ⟹ Q

En el coloreo, el invariante I2 (colores-usados ≤ Δ+1) es la hipótesis inductiva de la demostración clásica de χ(G) ≤ Δ+1. La verificación formal hace explícito lo que la inducción clásica hace implícitamente sobre el estado del algoritmo.
